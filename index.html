So how was useId called?

- First we create a type called UserItem that includes the attributes in it (id,userId,name,siteName,groupId,logoImage,contCategory,email,phoneNumber).
- Next, we create a type named UserSlice to inherit from UserItem and add setter functions (value setting functions) for each attribute of UserItem. Then we define another type named CDNSlice. It describes the structure of a part of the store related to CDN with 2 properties (cdnUrl and setCdnUrl ).
- Next we define a type named UserStoreSlice and it is a combination of UserSlice and CDNSlice.

Next we will define a type named StorePropsDevtools. It will describe the properties expected when creating a store with Zustand and enabling the devtool feature. And it will accept 2 parameters ( initializer, devtoolsOptions ). Initializer is a function of type StateCreator, which is a function provided by Zustand to initialize the initial state of the store. DevtoolsOptions is an optional parameter of type DevtoolsOptions and it is an interface imported from the zustand/middleware library.

- Then in the UserStore folder, we will create a store named UseUser and use the create function from Zustand. Inside, use the middleware in order of priority:
+ devtools: Enable the Devtools feature for the store, allowing easy monitoring and debugging of status.
+ persist : Use persist middleware to store store state.
+ Configure storage using createJSONStorage with sessionStorage to store state on the open tab. + The store name is set as "store".
- Next, the createUserSlice and createCDNSlice functions are used to create separate sections (slices) of the state. This helps separate logic and easily manage different parts of state related to users and CDN.
- Create a Hook named useUserStore : + This hook allows retrieving a specific set of properties from the useUser store. + It works similar to the useUserStore hook explained earlier.
- Finally export the useUser hook to the outside. This is the main hook to access and update the entire state of the store. Export the useUserStore hook to retrieve specific properties from the store.

- On the Login page, we will create an interface so users can log in. Here there will be a login button, right there we will create an onclick event. In that event we will create a function called handleClickLogin.

In the handleClickLogin function, we will use the MdmService.login(request) function to call the api with the POST method from the Mdm folder with file index.ts
